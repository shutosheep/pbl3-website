PBL3 GroupI Website

System Architecture Diagram

The system we created is called Vendeye, and it's a website for finding vending machines in Minamikusatsu. The system is divided into six parts. The access layer is the PC. The UI used by the system is Html and CSS. Its business layer is the main function of the website, including search, filter, geolocation, and random selection. The other is the data layer. As developers, we can upload data, store data, read and write data, and here we use the data set of vending machines that we collected and made ourselves. Finally, the website uses two APIs, one is open street map for displaying maps and positioning, and the other is our own Vendeye website.

Activity Diagram

This activity diagram shows the functional flow of Vending Machine Finder. The home page has three functions, two of which need to be connected to the Internet, which are to find vending machines by user location and to find vending machines by product. The other one does not need to be networked and is a random selection function.

The process for using each function is as follows:

  • Locate the vending machine according to the user's location: identify the user's location, and check whether it is connected to the Internet. If the Internet is successful, it displays the location of the vending machine closest to the user's location, displays its menu (including products, prices, etc.), and displays the recommended path to the vending machine. Exit after use (i.e., close the page).
  • Find vending machine by product: The user enters the name of the product to search, if the network is successful, it displays the location of the vending machine selling the product and displays its menu and recommended path. Exit after use (i.e., close the page).
  • Random selection: Open the random selection page, click the Start button, and the system will randomly select a product for the user, and get the location, menu, and recommended path of the vending machine.

    • Use Case Diagram

    The diagram shows the Vending Machine Finder system architecture, which is divided into three parts: users, features, and API. In our Vending Machine Finder system, users can use all the features including identifying the location, entering the product name, and using random selection to get the vending machine location, menu, and best path. These features currently use our self-created Vending Machine Finder API, our self-collected vending machine data set, and the Open Street Map API.
